Module five. Forms

Елемент form

Елемент form — це розділ HTML-документа, що містить інтерактивні елементи керування, які дозволяють користувачеві надсилати інформацію на веб-сервер. Семантично — це контейнер для групи пов'язаних елементів. Наприклад, форма зворотного зв’язку, реєстрації, оплати замовлення в інтернет-магазині. Кожен із цих компонентів інтерфейсу складається з групи інтерактивних елементів, згрупованих формою.

В елементі form немає обов'язкових атрибутів. Наведемо в прикладі кілька корисних необов'язкових:

  • name — ім'я форми, що є унікальним на поточній веб-сторінці. Може містити символи англійського алфавіту в будь-якому регістрі, цифри, підкреслення та тире. Не можна використовувати пробіл. Використовується як на сервері, так i на клієнті при роботі з формою.
  • autocomplete — визначає, чи може браузер автоматично заповнювати значення всіх елементів форми. Має всього два значення off i on.
  • novalidate — атрибут-прапор, не має значення. Говорить браузеру не перевіряти валідність введених даних під час відправки форми. Якщо атрибут не вказано, виконується вбудована у браузер валідація.

Відправлення форми

Клікаючи на <button type="submit"> або натискаючи клавішу Enter у будь-якому полі форми, ми можемо відправити форму. Ці дії також призводять до оновлення поточної вкладки браузера. Така поведінка є нормальною поведінкою за замовчуванням, яку можна змінити за допомогою мови програмування JavaScript.

Кнопка не успадковує багато стилів оформлення тексту. Основні — це сімейство шрифтів i колір тексту. Рекомендується встановити глобальний стиль тегу button. Інші властивості та їх значення варто брати з макета.

Елемент input

Елемент input — це універсальний елемент для створення різноманітних полів вводу.

Тип поля визначається атрибутом type.

Значення атрибуту type за замовчуванням: text. Його використання створює однорядкове текстове поле, що приймає будь-які символи.

За замовчуванням поле введення є рядковим елементом, який не успадковує сімейство шрифтів. Отже, до глобальних стилів можна додати правило для всіх полів вводу.

Концептуально input представляє запис у словнику, який відправляється на сервер.

Атрибут name визначає ім'я цього запису, а значення — це інформація, що ввів користувач. Ім'я поля має бути унікальним усередині форми. Значення атрибуту name також використовується при роботі з формою в JavaScript.

Елемент label

Елемент label пов'язує опис (мітку) з інтерактивним елементом форми. Текст i елемент пов'язані не тільки візуально, але й за логікою. Наприклад, скрінрідер оголосить текст мітки, коли користувач поставить фокус на пов'язаний елемент. При натисканні на мітку пов'язаний елемент автоматично отримає ефект фокусування.

Якщо елемент форми вкладається в label, зв'язок створюється браузером автоматично. Клік за текстом «Username» поставить фокус у поле введення.

Якщо елемент форми не вкладено в label, необхідно явно зв'язати їх через:

  • атрибут id елемента
  • атрибут for мітки

Атрибут placeholder

Атрибут placeholder дозволяє відображати текст-підказку про дані, які необхідно ввести в поле. Текст-підказка відображається, коли елемент input пустий та зникає при введенні тексту від користувача.

Атрибут placeholder можна використовувати в будь-якому елементі форми, де є текстове введення. Текст-підказка не замінює гарний label, а доповнює його прикладом на зразок даних, які слід вводити.

Псевдоелемент ::placeholder

Для оформлення тексту підказки використовується псевдоелемент ::placeholder.

Змінити стилі тексту підказки при наведенні миші або фокусі поля введення можна селекторами стану.

Псевдоклас :placeholder-shown

Псевдоклас :placeholder-shown застосовується залежно від видимості тексту-підказки та значення атрибута placeholder.

Він дозволяє налаштовувати властивості поля вводу під час відображення тексту-підказки. Слід пам’ятати, що як тільки користувач ввів хоча б один символ, підказка зникає.

Атрибут autofocus

За наявності цього атрибута, поле вводу автоматично отримує фокус при завантаженні. Отже, в ньому відразу можна буде набирати текст. За замовчуванням цього атрибута немає в жодному полі.

Фокус не може бути в декількох елементів одночасно. Отже, атрибут задається полю, з якого користувач має почати заповнювати форму. Атрибут autofocus — логічний, тобто без значення, вказується тільки назва.

Псевдоклас :focus-within

Застосовується до елемента, щойно він сам або елементи всередині нього отримують фокус. Це дозволяє застосувати стилі на:

  • мітку
  • форму
  • окреме поле форми при взаємодії користувача з полями форми
